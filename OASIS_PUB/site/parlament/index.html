Plan en 3 etapas

A / HANDS-IN
Arranca la sesión
  • El pub o un cliente autorizado extrae snapshot_in.
  • Se abre una Room layer2: inhabitants, master y capabilities.
  • Arrakis BOE publica el primer bloque: objetivo, reglas, criterios.
  • Firehoses quedan conectados pero todavía no deciden nada.
B / HANDS-ON
Se produce la capa de trabajo
  • Participantes hablan: Room, Telegram, Twitch, Oasis, bots.
  • Firehoses capturan; la sesión selecciona qué pasa a pieza.
  • Future-machine solidifica: vector → corpus → grafo → universo.
  • El teatro edita producción, corto o ejecución derivada.
C / HANDS-OFF
Cierre y regreso a Oasis
  • Se calcula summary_out: hashes, fuentes, firmas, decisiones.
  • Decoherence valida BOE ↔ conversación ↔ artefactos.
  • Network publica/fusiona como scriptorium-boe.
  • Si toca gobierno: vuelve como tribe-content, proposal o votes.

● Inhabitants → Layer2 (firehose)

  • firehose.atproto.raw · Jetstream filtrado.
  • bothub.telegram.msg · BotHub/IACM bridge.
  • room.message · Socket.IO ROOM_MESSAGE.

◀ Layer2 → Main (summary_out)

  • scriptorium.boe.append · bloque firmado append-only.
  • oasis.parliament.proposal · propuesta derivada.
  • oasis.tribe.content · pieza cristalizada (corto, voz).
  • oasis.parliament.vote · voto si la sesión lo emite.

○ Layer2 → Layer2 (interno, efímero)

  • session.cursor.tick · avance de cursor por firehose.
  • conversation.piece.accept · selección con evidenceHash.
  • future.machine.step · vector → corpus → grafo → universo.
  • decoherence.check · validación BOE ↔ artefactos.

Radicoma permanente vs sesión efímera

TTL = ∞ main · radicoma

Oasis / SSB

  • Append-only, P2P, sin borrado.
  • Ciclos lentos: 2 meses elecciones, ventanas de propuestas, leyes con fecha.
  • Rastros que sí persisten: scriptorium-boe, tribe-content, proposal, vote, law.
  • Reloj: días · semanas · gossip propagation.
TTL = sesión layer2 · pub volátil

Scriptorium Room

  • Volatile-gossip: ROOM_MESSAGE, buffers, cursores, capabilities.
  • Ciclos rápidos: segundos · minutos · una tarde.
  • Lo que no cristaliza se evapora con la sesión.
  • Reloj: tick de Socket.IO + cursor de firehose.
solo lo que pasa por BOE + summary_out + firma sobrevive al cierre de la sesión
